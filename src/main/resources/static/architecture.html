Overview

Story of a Lifetime is a multi-agent AI web application. Given a fixed start state, a fixed end state, and a human-provided list of facts, the system generates a plausible life story connecting them.

When new facts are added, the agent pipeline replans and produces a revised story — making visible exactly where and why the narrative diverged. The project demonstrates core AI Agent concepts: goal-directed planning, world model management, multi-agent coordination, a critic-driven feedback loop, and environment interaction through an append-only fact file.

Agents

Planner

Reads the world model and produces a structured 5–7 milestone outline. On retry attempts, the Critic's rejection reason is injected so the Planner knows exactly what to fix.

Default: claude-sonnet-4-6 · temp 0.3

Critic

Evaluates the Planner's outline for timeline plausibility, fact consistency, and logical gaps. Returns APPROVED or REJECTED with a specific reason. Max 3 attempts before falling back to last outline.

Default: claude-sonnet-4-6 · temp 0.1

Writer

Takes the approved outline and generates full prose. Never sees a rejected outline. Higher temperature produces more creative, varied storytelling.

Default: claude-opus-4-6 · temp 0.8

Explainer

Runs automatically after v2 is generated. Compares both stories, identifies what changed, and explains which new facts caused each divergence.

Default: claude-sonnet-4-6 · temp 0.3

Key design decisions

Facts are append-only

Once written, facts cannot be changed or removed. The agent must reconcile new facts with all previous ones — mirroring how real environments work. This is the core constraint that makes the v1 vs v2 comparison meaningful.

Agents have single responsibilities

Planner never writes prose. Writer never judges feasibility. Critic never generates content. Clean separation makes each agent inspectable, testable, and independently tunable.

The Critic loop is what makes it agentic

A pipeline executes in sequence. An agent evaluates and decides its next step. The Critic's reject-and-replan cycle — with the rejection reason fed back into the Planner — is what crosses that line.

Config and prompts live outside code

Model choice, temperature, story length, and all prompt wording are in files. You can experiment with any of these without recompiling — making iteration fast and the system transparent.

Cost tracked per agent

Every API call returns token usage. Each agent accumulates its own cost independently — making the tradeoffs between model choices visible and measurable in the UI on every request.
